Welcome to our community!


GiLit is a web-based platform for people to give and get help within their local community.


How it works
  • Prepare database in the first run
  • Register for a free account
  • Find your local community
  • Help out and gain points
  • Use your points to ask for help, or give them to others in need!
  • Extra: Meet up with the people who helped you or whom you helped

GiLit is created by Khanh Nghiem (Connecticut College '19). The name is short for "Give a Little, Get a Little", a brilliant phrase that Jason Karos (Connecticut College '18) came up with. Khanh wants to use software engineering to foster community engagement and spread kindness in the world. This is the prototype version and will be graded as Khanh's final project for his class COM214 - Web Technology and Mobile Computing. More features to come!

Objective of the web site

Gilit provides a platform to utilize human resources in a local community, allowing people to give and receive help without monetary incentives. It aims to foster community engagement and spread kindness in the world.

Description of the web site and target audience.

The website asks the user to register for an account and choose a local community to join. If the user already has a community, s/he can look at the Community Board. The Community Board has all the posts from the members of the community. Each community has at least 1 admin. Right now, Gilit does not allow registration for new admin account due to verification difficulty. Each user starts out with 0 points, and can gain points by commiting to help. Future plan: Only getting points when task marked as completed. If the user has more than 0 point, s/he can post on the community board and ask for help.

The current target audience of Gilit is the Conn Coll community.

Description of design considerations and aesthetics.

The design is minimalist, using the Bootstrap framework for mobile device friendliness. Navigation is simple. The experience in this prototype is quite linear.

Nature of information exchange with the server.

There are 3 databases - users, coms, and posts - all connected by integer id keys. All content is dynamically pulled from the database.

Operations performed on the server.

To pick a local community, the database takes the geolocation of the user and the lat and long of the community to compute the distance in miles. The database return the nearest community. Gilit also uses other basic operations on a MySQL databased: SELECT, INSERT, UPDATE...
